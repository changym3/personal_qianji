# 迭代记录

## v1.0 — 项目初始化与基础功能

- 使用 Vite + React 18 + TypeScript 搭建项目脚手架
- 安装核心依赖：Zustand、Recharts、Ant Design、uuid
- 实现 Zustand store + LocalStorage 持久化，定义 Account/Category/Group/Item 数据模型
- 实现资产负债表主界面：分组展示、条目增删改、金额编辑
- 使用 Recharts 实现按周的净资产/总资产/总负债趋势折线图
- 响应式适配、数据导入导出（JSON）、数字格式化、空状态处理

## v1.17 — 多账本、当前资产 Tab、配置记忆与界面微调

- **多账本**：`useAccountStore` 支持按账本名存储多套 `Account`（`accountsByLedger` / `ledgerNames` / `currentLedgerName`），默认账本名为「测试账本」；旧版仅含单一 `account` 的 LocalStorage 数据会自动迁移到该账本
- **账本下拉**：顶栏以 `LedgerDropdown` 替代纯 Select，在下拉内支持新增账本、重命名、删除（至少保留一个账本）；新增账本后自动切换到该账本并加载其独立数据
- **「当前资产」Tab**：原「资产负债表」Tab 更名为「当前资产」，进入后展示与「资产时间线」最新一张卡片点击后相同的页面（`SnapshotEditor` 资产卡片详情）；无快照时提示先新增快照
- **趋势页配置记忆**：新增 `useChartSettingsStore`（Zustand + persist），持久化保存聚合方式、金额单位、趋势指标、时间范围预设，刷新后沿用上次选择（自定义起止日期仍为会话内状态，不写入持久化）
- **文案与布局**：趋势页顶部「全局配置」更名为「配置」；移除原标题旁的说明提示；在「聚合方式」旁增加悬浮说明（按周/按月时仅保留周期内最后一次快照）
- **自定义时间范围**：选择「自定义」时，`RangePicker` 紧挨「时间范围」一行展示在配置卡片内部，不再单独落在卡片外



## v1.18 — 条目"不计入总资产"标记

- `Item` 类型新增可选字段 `excludeFromTotal?: boolean`

- 分组小计也同步排除 `excludeFromTotal` 条目，仅展示实际计入金额

## v1.19 — 资产卡片详情页三段式结构与快照备注

- **资产卡片详情页**（`SnapshotEditor`）改为三大块结构：资产、负债、备注
- 资产与负债大块标题行可点击整块收起/展开，标题右侧实时显示对应总金额（资产绿色、负债橙色）
- 新增「备注」区块，位于资产/负债大块下方，平时直接展示文本（无内容时显示灰色占位提示），点击后切换为多行文本编辑框，失焦后自动回到展示态
- `WeeklySnapshot` 类型新增可选字段 `remark?: string`
- `saveFullSnapshot` 新增第三个参数 `remark?: string`，保存时写入快照
- `TrendChart` 的 `SnapshotSummary` 接口同步增加 `remark` 字段，打开编辑页时一并传入
- `SnapshotDetail`（只读详情页）同步改造：资产/负债大块可折叠，底部新增备注展示区

## v1.20 — 全局设置：金额单位 & 家庭资产模式

- **金额单位全局化**：`unit`（元/万）从趋势页配置区移出，提升为全局设置，显示在 Tab 栏右侧，对「趋势图表」和「当前资产」两个 Tab 同时生效
- **家庭资产计入模式全局化**：`includeFamilyAssets` 布尔值升级为 `FamilyAssetMode` 三态枚举（`exclude` 不计入 / `as-liability` 计入负债 / `deduct-asset` 从资产扣除），同样提升为全局设置
- **两种计入方式语义**：「计入负债」将家庭资产总额加入 `totalLiabilities`；「从资产扣除」将其从 `totalAssets` 中减去；两种方式净资产结果相同，区别仅在资产/负债的分配
- **SnapshotEditor 联动**：资产卡片详情页中所有金额展示统一读取全局 `unit`；汇总数值（净资产/总资产/总负债）按全局 `familyAssetMode` 动态计算
- **TrendChart 联动**：新增 `calcSnapshotTotals` 函数，图表数据与快照卡片均按 `familyAssetMode` 从快照 `categories` 动态重算，实时响应模式切换
- **UI 控件**：Tab 栏右侧两个下拉按钮替代原 Switch/Segmented，显示当前值，点击弹出选项菜单；从趋势页「配置」区移除对应旧控件

## v1.21 — 家庭资产独立大类

- **家庭资产升级为独立大类**：与「总资产」「总负债」并列，拥有完全相同的编辑能力（添加/删除/重命名分组，条目增删改，条目拖拽排序，分组排序）
- **数据分离**：编辑器新增 `familyGroups` state，初始化时从 `assetGroups` 中过滤 `isFamilyAsset=true` 的条目独立管理；保存时合并回 asset category，数据格式完全向后兼容
- **拖拽扩展**：`SectionType` 扩展支持 `'family'`，条目可在资产/负债/家庭资产三大类之间跨类拖拽；拖入家庭资产区自动标记 `isFamilyAsset: true`，拖出时自动清除
- **条目图标**：家庭资产条目使用 🏠 图标，与资产（📊）负债（📋）区分
- **「计入家庭资产」开关**：所有大类的条目编辑行均显示该开关；家庭资产大类下默认开启，可手动取消（取消后条目保存为普通资产）；新增条目时点击「添加条目」会按所在大类自动预设开关状态
- **大类标题**：家庭资产大类标题行右侧显示计入模式标签（「计入负债」/「从资产扣除」），不计入时不显示
- **空白快照**：家庭资产大类也默认添加一个「未分组」分组（与资产/负债一致）

## v1.22 — 快照卡片体验优化

- **备注外显**：快照卡片详情页的备注内容显示在资产时间线卡片上，位于进度条下方，与「净资产」等文字对齐（相同横轴）；无备注时不占位
- **进度条修复**：将原来「总资产/（总资产+总负债）」的比例修正为「净资产/总资产」，蓝色代表净资产占比，橙色代表负债占比，真实反映财务健康度；负净资产时进度条不异常显示
- **UI 控件统一**：Tab 栏「金额单位」改为与「家庭资产」一致的下拉按钮风格；「金额单位：」标签文字显示在按钮左侧

## v1.23 — 新建快照安全性修复

- **默认日期避让**：新建快照时，默认日期从今天开始向前查找，自动跳过已有快照占用的日期，避免默认日期恰好与最新快照日期相同导致数据被覆盖
- **保存时二次校验**：非编辑模式下保存时若检测到日期与已有快照冲突，弹出警告提示修改日期，不再静默覆盖已有快照

## v1.24 — Tab 栏与顶部导航固定

- **Tab 栏 sticky**：趋势图表、当前资产 Tab 标签及右侧「金额单位」「家庭资产」设置按钮在页面滚动时固定在顶部 Header 下方，始终可见；通过 `tabBarStyle` 设置 `position: sticky`、`top: 64`（紧贴 Header）、`zIndex: 99` 及与页面背景色一致的背景实现
